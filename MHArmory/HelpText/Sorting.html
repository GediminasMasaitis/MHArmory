Sorting To sort armor set search results, first click the Sort results... button at top right of the main window.
This opens the sort window. This window is made of two part, the left pane and the right pane.

The left pane

There, you can add sorting logic simply by clicking the Add sorting logic button.
Details comming in the next section.
You can also:
  • Make a sorting logic the default one when searching, by clicking the Make active button.
    This will also sort the current search result with this sorting logic immediately.
  • Sort the current search result with a given logic without making it active, by clicking the Apply button.
The number between parenthesis beside the sorting logic name is the number of sort criteria it contains.
The active sort logic is indicated with blue and bold text on its name.

You can also edit existing sorting logics by clicking the Edit mode button and do:
  • Delete existing sorting logics by clicking the X button
  • Renamed existing sorting logics by simply typing a new name in the text area
  • Move them up or down to reorganize them, by click the the small vertically aligned buttons beside the delete button

The right pane

To use the right pane, you must first select a sorting logic by clicking it.
Then, at top of the right pane, a button named Add sort criteria should appear.
Here you can add as many sort criteria as you want, they are all chained to produce a single sort when searching armor sets.
As for the sorting logics, you can delete some by clicking the X button, and reorganize them by clicking the small vertically aligned buttons.

Selecting multiple search criteria means, for result that are considered equal on the first sort criteria, then the next will sort them on a different criteria, and so on.
For example, let's assume you searched and that produced 4 results:
  • Defense: 20, Spare slots: 5
  • Defense: 30, Spare slots: 3
  • Defense: 30, Spare slots: 4
  • Defense: 40, Spare slots: 3

If you add only a Defense sort criteria, this will produce the following:
  1. Defense: 40, Spare slots: 3
  2. Defense: 30, Spare slots: 3
  3. Defense: 30, Spare slots: 4
  4. Defense: 20, Spare slots: 5

Note that the second and third results have their spare slots not taken into account, and order hasn't changed compared to the original order.

Now let's try to sort by Defense, then by Spare slots, we should get the following result:
  1. Defense: 40, Spare slots: 3
  2. Defense: 30, Spare slots: 4
  3. Defense: 30, Spare slots: 3
  4. Defense: 20, Spare slots: 5

Now second and third results are in a better order.
That means, for results that were considered equal on Defense, the Spare slots sorting criteria added a level of granularity to produced an even better sorting.

One last example to help you understand the order of sorting: now sorting by Spare slots, then by Defense
  1. Defense: 20, Spare slots: 5
  2. Defense: 30, Spare slots: 4
  3. Defense: 40, Spare slots: 3
  4. Defense: 30, Spare slots: 3

Note that now the Spare slots is the main criteria, and Defense is a refinement among equal Spare slots results.
